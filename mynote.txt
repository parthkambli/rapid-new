3. Special Notes
For Long Content Handling:

Authorized Person Section: This can be long with multiple departments. The class authorized-person-section allows it to break if needed, but individual department items (dept-list-item) won't break mid-line.
Long Addresses: The long-text-field class enables natural word wrapping with hyphens, so addresses won't create awkward gaps.
Declaration Section: If your form has a lot of content before it, the declaration might need to start on a fresh page. The current CSS allows this naturally. If you want to FORCE it to always start on a new page, change the declaration div to:

jsx   <div className="page-break-before declaration-section mt-6 text-center">

Signature Grid: Always kept together with signature-section class. If you have both Doctor 1 and Doctor 2 signatures, they'll stay on the same page.
Grid Rows: Individual grid items (each <div> inside .grid-compact) won't break, but the grid itself can flow across pages if needed. This prevents awkward half-rows.

Browser Compatibility:

Uses both page-break-* (older browsers) and break-* (modern standard)
Tested for Chrome, Firefox, Edge print rendering
-webkit-print-color-adjust: exact ensures background colors print (section titles)

Testing Tips:

Print Preview (Ctrl+P / Cmd+P) to see actual layout
Test with different data lengths (short vs long addresses, many departments, etc.)
Check that no single line is orphaned at bottom of page
Verify signatures always appear together
